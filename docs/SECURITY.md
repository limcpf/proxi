# 보안 가드레일

## 기본 원칙
- 비밀정보, 토큰, 키 파일은 저장소에 커밋하지 않는다.
- 사용자가 명시하지 않은 신규 의존성 추가는 기본적으로 금지한다.
- 외부 네트워크, 외부 저장소, 실제 데이터에 영향을 줄 수 있는 작업은 필요 시 승인 절차를 따른다.

## 변경 시 추가 확인이 필요한 영역
- 인증, 인가, 세션 처리
- 개인정보 또는 민감한 운영 데이터
- 외부 API 연동과 웹훅
- DB 스키마, 마이그레이션, 데이터 백필

## 문서화 규칙
- 보안 경계가 바뀌면 설계 문서 또는 실행 계획에 위험과 완화책을 남긴다.
- 검증 공백이 있으면 `QUALITY_SCORE.md` 또는 기술 부채 문서와 연결한다.
