# 실행 계획 운영 규약

## 계획 문서가 필요한 경우
- 작업이 여러 단계로 나뉜다.
- 중단 후 재개 가능해야 한다.
- 검증 절차를 따로 추적해야 한다.
- 구조 변경, 데이터 변경, 운영 영향이 있다.

## 저장 위치
- 진행 중인 계획: `docs/exec-plans/active/`
- 완료된 계획: `docs/exec-plans/completed/`

## 권장 섹션
- `목표`
- `범위`
- `작업 단계`
- `검증 방법`
- `결정 로그`
- `남은 이슈`

## 운영 규칙
- 작은 단건 수정까지 모두 계획 문서로 만들지 않는다.
- 상태가 바뀌면 같은 날 바로 갱신한다.
- 문서 라우팅 대상 파일을 추가, 이동, 완료 상태로 옮기면 관련 `index.md`/`README.md` 와 `docs/generated/context-map.json` 을 함께 갱신한다.
- 완료 후에는 결과와 후속 과제를 남기고 `completed/`로 이동한다.
