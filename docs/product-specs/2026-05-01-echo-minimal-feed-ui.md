# Echo 미니멀 피드 UI 요구사항

## 목표
- Echo 피드 화면을 짧은 생각을 즉시 남기고 최근 Echo 본문만 빠르게 훑는 화면으로 줄인다.
- 첫 방문 설명보다 반복 사용자의 작성과 읽기 속도를 우선한다.
- 기존 작성, 첨부, 검색, 아카이브, 상세 보기 기능은 유지하되 기본 화면에서 보이는 정보와 액션을 최소화한다.

## 현재 문제
- 상단 hero 와 설명 문구가 반복 사용 화면에서 실제 행동보다 큰 공간을 차지한다.
- `ECHO`, `FEED`, `최근 Echo`, 앱 이름처럼 같은 맥락의 라벨이 반복된다.
- 작성 카드가 라벨, 첨부 input, 큰 padding 때문에 실제 입력 기능보다 무겁게 보인다.
- 검색, 아카이브, 표시 개수, 상세 보기, 댓글 없음, 본문 복사가 모두 기본 노출되어 본문보다 부가 기능이 눈에 띈다.
- 브라우저 기본 파일 input 이 시각 체계와 맞지 않고 미니멀 화면의 노이즈가 된다.

## 사용자 경험 기준
- 화면의 중심 목적은 `지금 떠오른 생각을 남기고 최근 Echo 를 읽는다` 이다.
- 첫 화면에서 작성창과 최근 피드 첫 항목이 함께 보여야 한다.
- 작성 영역은 별도 제목 없이 본문 입력, 접힌 첨부 진입점, 저장 버튼만 기본 노출한다.
- 피드는 본문을 최우선으로 보여주고 작성자, 시간, 댓글 수, 상세 이동, 복사는 낮은 대비의 보조 정보로 둔다.
- 검색과 첨부는 필요할 때만 열리는 보조 기능이다.
- 아카이브 이동은 유지하지만 피드의 주요 행동처럼 보이면 안 된다.

## 화면 변경 기준
- 제거: hero title, hero 설명, `ECHO` kicker, `FEED` kicker, 반복 section heading.
- 축소: page top padding, compose panel padding, compose 와 feed 사이 간격, feed card padding.
- 유지: masthead 의 앱 이름과 기본 navigation, Echo 작성, 파일 첨부, 검색, 아카이브 이동, 상세 보기, 본문 복사.
- 기본 숨김: 검색 input, 파일 input.
- 조건부 노출: 검색어가 있거나 사용자가 `검색`을 누른 경우 검색 input 과 초기화 액션을 보인다.
- 조건부 노출: 첨부 파일을 선택한 경우 파일명 chip 또는 caption 을 보인다.

## 수용 기준
- 피드 route 에서 hero 문구 없이 compose 가 첫 번째 주요 surface 로 보인다.
- create composer 에서 시각적으로 보이는 label 은 제거하고 textarea placeholder 로 작성 목적을 전달한다.
- 파일 선택 전에는 브라우저 기본 파일 input 이 화면에 직접 보이지 않는다.
- `첨부` 보조 액션으로 파일 선택이 가능하고, 선택된 파일명은 저장 전 확인할 수 있다.
- 검색어가 없을 때 검색창은 기본 노출되지 않고 `검색` 보조 액션만 보인다.
- 검색어가 있을 때는 검색창, 검색 실행, 초기화가 보여 현재 검색 상태를 이해할 수 있다.
- 최근 피드 header 는 `최근` 과 표시 개수 정도만 남긴다.
- Echo card 에서는 본문이 metadata 와 action 보다 먼저 읽힌다.
- `상세 보기`와 `본문 복사`는 primary button 처럼 보이지 않는다.
- loading, empty, error, disabled 상태 문구는 유지하되 화면 전체를 밀어내는 큰 panel 로 만들지 않는다.
- 상세와 아카이브 route 도 메인 피드와 같은 Carbon-lite shell, compact surface, divider list 밀도를 따른다.
- 상세 route 에서 본문, 댓글 없음 상태, 댓글 composer 는 내용 높이 중심으로 배치되고 큰 빈 카드처럼 보이지 않는다.
- 아카이브 route 는 큰 hero 없이 compact header, 접힌 검색, divider 목록으로 표시된다.
- Echo 본문은 섹션 라벨, metadata, count, status, placeholder 보다 더 높은 대비와 큰 body scale 로 보여야 한다.
- `피드`, `댓글`, `댓글 없음`, 날짜, 작성자, empty 안내 같은 정보성 문구는 본문처럼 읽히지 않도록 낮은 위계로 표시한다.
- `상세`, `복사`, `검색`, `아카이브`, `첨부` 같은 시스템 액션 문구는 link/button element 차이 없이 같은 크기와 굵기로 보인다.
- 시스템 문구는 본문보다 낮은 대비를 유지하되, 너무 작거나 약하지 않게 14-16px semibold 계열로 읽힌다.

## 범위 밖
- Echo API, shared 계약, 백엔드, DB schema 는 변경하지 않는다.
- attachment 저장 방식과 다운로드 URL 정책은 변경하지 않는다.
- 새 dependency, 새 전역 상태 라이브러리, 새 디자인 token 은 추가하지 않는다.
