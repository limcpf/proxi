# 문서 라우팅 매니페스트 도입

상태: `accepted`

## 배경
- 루트 `AGENTS.md` 와 `docs/` 인덱스는 있었지만, 에이전트가 문서 구조를 같은 방식으로 읽는 기계용 기준은 없었다.
- 링크 깨짐, 인덱스 누락, 폴더 책임 위반은 수동 점검에 의존해 PR 단계에서 쉽게 놓칠 수 있었다.
- `docs/generated/` 는 비어 있었고, 기계가 읽을 수 있는 문서 라우팅 목차가 없었다.

## 결정
- `docs/generated/context-map.json` 을 문서 경로, 책임, 읽기 조건을 담는 기계용 기준으로 추가한다.
- `pnpm run verify:docs` 는 `context-map.json`, 로컬 링크, 인덱스/`README`, `docs/generated/` 경계를 함께 검증한다.
- PR 검증은 `pnpm run verify` 를 단일 진입점으로 사용하고, 문서 구조 위반을 실패로 처리한다.

## 대안
- Markdown 인덱스만 기준으로 두는 방식은 기계 검증 규칙을 안정적으로 표현하기 어려워 채택하지 않았다.
- 루트 `AGENTS.md` 하나만 기준으로 두는 방식은 폴더별 책임과 장기 작업 기록을 표현하기 어려워 채택하지 않았다.

## 영향
- 문서를 추가, 이동, 삭제하면 관련 `index.md`/`README.md` 와 `context-map.json` 을 함께 갱신해야 한다.
- 에이전트는 문서 구조 탐색이 필요할 때 `context-map.json` 을 먼저 읽어 필요한 문서만 좁힐 수 있다.
- CI 는 링크 깨짐과 인덱스 누락을 PR 단계에서 차단한다.

## 후속 작업
- 필요하면 `context-map.json` 동기화 스크립트 또는 템플릿 자동화를 별도 설계로 분리한다.
