# 에이전트 운영 문서 시스템 도입

상태: `accepted`

## 배경
- 루트 `AGENTS.md` 는 기본 규칙만 있었고, 프로젝트별 세부 라우팅과 작업 기억 구조가 거의 없었다.
- `apps/back/AGENTS.md` 는 비어 있어서 백엔드 변경 시 읽어야 할 기준 문서를 안내하지 못했다.
- 설계 기록, 실행 계획, 품질 기준을 분리해 저장할 위치가 없어서 대화 문맥에 의존할 위험이 있었다.

## 결정
- 루트 `AGENTS.md` 는 짧은 인덱스/라우터로 유지한다.
- 세부 규칙과 작업 기억은 `docs/` 아래에 구조화한다.
- 프로젝트별 `AGENTS.md` 는 해당 프로젝트 전용 라우터 역할만 맡는다.
- 장시간 작업은 `docs/exec-plans/` 에 기록하고, 설계 결정은 `docs/design-docs/` 에 기록한다.

## 대안
- 모든 규칙을 루트 `AGENTS.md` 하나에 몰아넣는 방식은 문맥 낭비와 유지보수 비용이 커서 채택하지 않았다.
- 외부 위키나 메신저 기록을 기준으로 삼는 방식은 저장소 밖 지식에 의존하므로 채택하지 않았다.

## 영향
- 에이전트는 루트에서 짧게 라우팅 받고, 필요한 문서만 추가로 읽을 수 있다.
- 구조적 변경의 근거와 장시간 작업의 상태를 저장소 안에서 추적할 수 있다.
- 아직 자동화 검증은 없으므로 문서 최신성은 운영 규칙과 기술 부채로 관리한다.

## 후속 작업
- 문서 최신성 검사와 링크 검사를 자동화 후보로 검토한다.
- 새 프로젝트가 추가되면 같은 문서 계층을 복제한다.
